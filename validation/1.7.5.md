## Features validated in 1.7.5

- **Authorized bots (purple) on main dashboard chart** — 5th dataset `authorized bots` in `#bc8cff` purple, dashed, maps `b.authorized_robot` from timeline response.
- **Authorized bots (purple) on agents chart** — 4th dataset; `tickChart()` populates from `b.authorized_robot||0`; totals line shows `auth-bots=N`.
- **Authorized bots (purple) on geomap** — purple circle markers for `authorized_robot` count per cell; legend entry added; tooltip shows count; scrubber bucket merging tracks `ar` counter; `rebuildBuckets()` routes `'authorized_robot'` kind to `ar++`.
- **Backend: metrics_endpoint** — timeline buckets include `authorized_robot` field, extracted from `by_reason["authorized-robot"]` (in-memory defaultdict or DB JSON column). DB query now selects `by_reason` column.
- **Backend: agents_timeline_endpoint** — dedicated SQL query for `reason='authorized-robot'`; `authorized_robot` field in each series entry and totals dict.
- **Backend: geo_data_endpoint** — `authorized-robot` events classified as `kind='authorized_robot'` (not `'blocked'`), no longer inflates blocked count on map. All point and country init dicts include `authorized_robot: 0`.
