## Features validated in 1.7.6

- **Category filter bar** ‚Äî five pills (Allowed/Blocked/Missed/Auth Bots/GW Mgmt) on `main.html` and `agents.html`. Active by default. Controls Chart.js dataset visibility and clients/suspects table rows simultaneously.
- **`_clientCats` / `_agentCats`** ‚Äî classify each entry by priority: `is_authorized_bot` ‚Üí `gwmgmt` (last_path prefix `/antibot-appsec-gateway/`) ‚Üí `blocked` ‚Üí `missed` (score ‚â• 20) ‚Üí `allowed`.
- **`_renderClientsTable(list)`** ‚Äî standalone function extracted from `tick()` so `_applyFilters()` can re-render the filtered table without a network request.
- **GW Mgmt filter** ‚Äî captures entries whose `last_path` starts with `/antibot-appsec-gateway/`; table-only (no corresponding chart dataset).
- **Bug: auth bot priority over gwmgmt** ‚Äî `is_authorized_bot` now checked before the gwmgmt path check; auth bots polling the health endpoint no longer vanish from the Auth Bots filter.
- **Bug: auth bots bypassing `min_score` gate in `agents_data_endpoint`** ‚Äî `_s_is_auth_bot` hoisted before `if score < min_score: continue`; gate guarded as `and not _s_is_auth_bot`. Auth bots now always appear in the agents suspects table.
- **Bug: null comps/mets for score-0 auth bots** ‚Äî safe fallback dicts added when `_s_is_auth_bot and not comps/mets`.
