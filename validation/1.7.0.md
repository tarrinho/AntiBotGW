| Failed | 0 |
| Notes  | One regression introduced then fixed this session: `test_take_token_burst_then_block` failed when run after `test_165_every_knob_persists_round_trip` due to cross-contamination between the importlib-loaded proxy (module_1, used by test_critical) and the session-scoped proxy fixture (module_2, used by test_async). Root cause: `db_load_config`'s propagation loop updated `rate_limit.RATE_LIMIT_BURST=88` via the shared sys.modules, while test_165's restore (`setattr(module_1, k, v)`) only restored module_1. Fix: removed the sys.modules propagation loop from `db/sqlite.py`; `proxy.py`'s `db_load_config()` wrapper now cascades via `_ProxyModule.__setattr__` only when the caller is the registered module (`globals() is sys.modules["proxy"].__dict__`), which is never true in the importlib-loaded test context. |
