## Changes in this version

- **Top Paths filtered by category pills** — `by_path_by_cat` added to `state.py` (one `defaultdict(int)` per category); incremented in `core/metrics.py record()` alongside `events_by_cat` using the same priority classification (gwmgmt > authbots > ban > missed > allowed). `metrics_endpoint` in `core/proxy_handler.py` uses the merged `by_path_by_cat` subset when `cats` param selects a subset; falls back to `metrics["by_path"]` when all five are active.
- **Bidirectional chart legend ↔ filter pill sync** — `_toggleCatFilter(cats)` shared JS function in `main.html`; chart `plugins.legend.onClick` delegates to it via `_DS_CATS` map. All three filter surfaces (top pills, chart legend, panel mini-legends) kept in sync through `_applyFilters()` → `_syncPanelLegends()`.
- **Panel mini-legends on Clients, Top Paths, Live Events** — `.panel-legend` / `.panel-leg-item` spans added to each panel h2; click calls `_toggleCatFilter()`; dim at 28% opacity when inactive; `_syncPanelLegends()` syncs on every `_applyFilters()` call.
- **`status_endpoint` Cache-Control fix** — `core/proxy_handler.py` `status_endpoint` now returns `Cache-Control: no-store` (was the only admin endpoint missing it).
- **`tests/test_endpoints_dynamic.py`** (new) — 114-test live aiohttp integration suite covering all admin HTTP endpoints and cache-control headers.
