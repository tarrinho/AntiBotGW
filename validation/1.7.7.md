## Features validated in 1.7.7

### GW Mgmt filter fix (original)
`protect()` middleware now calls `record()` for authenticated operator requests before returning. Previously returned early without recording → gwmgmt filter showed zero entries.

### Geo dashboard loading/ready pill (this cycle)
Added `#load-status` CSS pill to `dashboards/geo.html`:
- Starts yellow with pulsing dot animation ("Loading") on page open
- Flips to solid green "Loading Ready" inside double `requestAnimationFrame` after first successful `tick()` data fetch (after `renderAsns()` completes)
- Matches the controls dashboard pattern exactly
- Image rebuilt as `appsec-antibot-gw:1.7.7` and deployed healthy

### Dashboard data test fixes (this cycle)
Three stale tests in `test_dashboard_data.py` updated to match current endpoint response shapes:
- `agents-data` key: `agents` → `suspects`
- `logs-data` key: `events` → `rows`
- `path-hits`: added required `?path=/` param; key `paths` → `ips`

### Dashboard chart bug fix (this cycle)
`main.html` tick() timeline block used `'blocked'` as the filter key for `datasets[2].hidden` — a category that never exists in `_activeFilters`. Corrected to `'ban'` / `'reallyban'` (matching `_applyFilters()`), so the Blocked line in the timeline chart is no longer unconditionally hidden after each data refresh.
