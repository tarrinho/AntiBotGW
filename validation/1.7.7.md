## Features validated in 1.7.7

- **GW Mgmt filter fix** — `protect()` middleware in `core/proxy_handler.py` now calls `record()` for authenticated operator requests to admin paths before returning. Previously the middleware returned early at `return await handler(request)` without recording, so every dashboard page load and API poll by a logged-in operator was invisible in `ip_state`. The `gwmgmt` filter pill on both dashboards showed zero entries. Fix: await the handler response, then call `record(ip, ua, path, status, "operator-passthrough")` before returning.

## Regression tests added (1.7.7)
