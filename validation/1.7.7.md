## Features validated in 1.7.7

### GW Mgmt filter fix
`protect()` middleware now calls `record()` for authenticated operator requests before returning. Previously returned early without recording → gwmgmt filter showed zero entries.

### Geo dashboard loading/ready pill
Added `#load-status` CSS pill to `dashboards/geo.html`:
- Starts yellow with pulsing dot animation ("Loading") on page open
- Flips to solid green "Loading Ready" inside double `requestAnimationFrame` after first successful `tick()` data fetch
- Matches the controls dashboard pattern exactly

### Dashboard data test fixes
Three stale tests in `test_dashboard_data.py` updated to match current endpoint response shapes:
- `agents-data` key: `agents` → `suspects`
- `logs-data` key: `events` → `rows`
- `path-hits`: added required `?path=/` param; key `paths` → `ips`

### Dashboard chart bug fix
`main.html` tick() timeline block used `'blocked'` as the filter key for `datasets[2].hidden` — corrected to `'ban'` / `'reallyban'` so blocked line no longer hidden unconditionally.

### BYPASS_PATHS audit trail
`protect()` bypass early-return now writes `("event", ..., "bypass-path")` to `db_queue` before returning. Every bypassed access is traceable in the events log and logs dashboard. `ip_state` intentionally stays empty (no bot scoring overhead).

### Main dashboard path search + event log panel
- Path search input in the filter bar filters clients table by `last_path` substring match
- "Path event log" panel (hidden when no query) fetches `/secured/logs-data?q=<path>` and renders all matching DB events including `bypass-path` entries (time, IP, path, status, reason, UA)
- Debounced 300 ms; clear button
