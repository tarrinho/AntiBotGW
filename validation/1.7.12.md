## Changes in this version

- **Import configuration "Test" button** — dedicated dry-run validation button in settings.html; fires `POST /__settings-import?dry_run=1` independently of the dry-run checkbox; result labelled `TEST — no changes applied`; `_doImport(dry)` helper refactored so both Test and Apply share the same logic.
- **DOMPurify output sanitisation** — `purify.min.js` (26 KB) self-hosted in `dashboards/assets/`; added to all 7 dashboard HTML files; 79 dynamic `innerHTML` assignments wrapped with `_dp()` helper (`DOMPurify.sanitize` with graceful fallback); addresses CSP `unsafe-inline` XSS risk via defense-in-depth.
- **7 new QA tests** (`TestSettingsImportTestButton`) — HTTP 200, `dry_run=true` in response, no state mutation, zero errors on valid ZIP, 400 on empty body, knob counting, HTML presence of `btn-test`.
- **Q2 deferred** — REFILL=0 div-by-zero; operator-misconfiguration edge case; defaults are positive; no real-world impact; tracked for 1.7.13.
