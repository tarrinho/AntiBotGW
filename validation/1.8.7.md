## Security fixes summary (v1.8.7)

### DET4-02 — Redirect maze dest binding

| | |
|---|---|
| Finding | `dest` parameter not bound in HMAC — attacker could swap destination on a valid maze token |
| Fix | `_dest_hash(dest)` (SHA-256[:16]) included in HMAC msg; `_verify_maze_token` requires dest |
| Files | `detection/redirect_maze.py` |
| Tests | `TestDET402MazeDestBinding` (8 tests) |

### DET4-03 — Interaction probe identity binding

| | |
|---|---|
| Finding | Probe token bound to IP — NAT/shared-IP allows token relay across different sessions |
| Fix | Token bound to `get_identity(request)` session key; both injector (`js_challenge.py`) and validator (`interaction_report_endpoint`) use identity |
| Files | `detection/interaction.py`, `challenge/js_challenge.py` |
| Tests | `TestDET403InteractionIdentityBinding` (5 tests) |

### DET4-04 — All-identical-timestamp bypass

| | |
|---|---|
| Finding | After P1-8 offset clamping, submitting all-zero offsets bypasses interaction check (all clamped to 0) |
| Fix | `max(ts_vals) == min(ts_vals)` with ≥5 event threshold → `no-interaction` verdict |
| Files | `detection/interaction.py` |
| Tests | `TestDET404IdenticalTimestampBypass` (5 tests) |

### PROXY4-01 — UPSTREAM hot-reload SSRF

| | |
|---|---|
| Finding | Hot-reload validator checked scheme/length only, never called `_assert_upstream_public()` → internal IP reachable via ENV update |
| Fix | `_upstream_safe_to_reload` wraps `vhost._assert_upstream_public`; `ALLOW_PRIVATE_UPSTREAM` removed from `_HOT_RELOAD_KNOBS` |
| Files | `core/proxy_handler.py` |
| Tests | `TestPROXY401UpstreamValidator` (8 tests) |

### PROXY4-02 — Host header injection in Location rewrite

| | |
|---|---|
| Finding | `client_host = request.host or up_parsed.netloc` used raw attacker-controlled Host header for redirect Location |
| Fix | Validate `request.host` against `ALLOWED_HOSTS` frozenset before use; fall back to `up_parsed.netloc` |
| Files | `core/proxy_handler.py` |
| Tests | `TestPROXY402ClientHostValidation` (4 tests) |

### PROXY4-03 — Module __setattr__ builtin overwrite

| | |
|---|---|
| Finding | `_ProxyModule.__setattr__` propagated all writes to all `sys.modules` without denylist — builtin names overwritable |
