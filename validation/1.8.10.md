### Known failure (pre-existing, not a product defect)
`test_165_every_knob_persists_round_trip` — round-trips every `_HOT_RELOAD_KNOBS`
entry, but `*_ENABLED=False` values leak across the `proxy` ⇄ `core.proxy_handler`
module split (its `finally` restores only one side; `db_load_config` is
overlay-only with no reset), disabling detection for later tests in the same
interpreter. Pre-dates this work (3 knobs — BOT_DETECTION_ENABLED, TRUST_XFF,
TRUSTED_PROXIES — were added to the registry without updating the test). Each
test file passes in isolation. Proper fix requires subprocess isolation — tracked
separately. Left as a documented known-fail rather than masked.

### Test alignment (ALLOW_PRIVATE_UPSTREAM)
`ALLOW_PRIVATE_UPSTREAM` is intentionally hot-reloadable + default-allow (operator
request; SSRF tradeoff documented in `core/proxy_handler.py`). Stale tests that
asserted the prior always-on / not-hot-reloadable behavior were updated to match,
while keeping the SSRF guard tested via its opt-in path (`test_v180`, `test_v187`,
`test_v185`, `test_endpoints_dynamic`).
