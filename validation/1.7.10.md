- **26 new QA tests** — covering normalizeId field mapping (both shapes), strict null checks on stealth/tokens, weighted/count label logic, `|| {}` crash guard, risk_score branch, ip-intel placeholder, `.toFixed(1)` formatting, escapeHtml on all user fields, byte-identical IIFE drift guard.

### §15b — Active DAST Probe

| Probe | Expected | Actual | Signal | Pass? |
|---|---|---|---|---|
| IDOR (`GET /api/users/1` + `X-User-Id: 2`) | flagged or blocked | 200, `ua-platform-mismatch` | score=30 | ⚠️ pass-through (application-level IDOR; WAF cannot detect without business context) |
| Broken object auth (`GET /api/orders/9999` + fake Bearer) | flagged or blocked | 200, `ua-platform-mismatch` | score=30 | ⚠️ pass-through (same — application-level) |
| Mass assignment (`POST /api/profile` with `role:admin`) | flagged or blocked | 201, `ua-platform-mismatch` | score=30 | ⚠️ pass-through (application-level) |
| Path traversal null byte (`/static/..%00/etc/passwd`) | blocked | **400** | gateway rejected null-byte URL | ✅ |
| XXE (`POST /api/upload` XML with `SYSTEM "file:///etc/passwd"`) | body-lfi or blocked | 404, `body-lfi` fired | score=80 | ✅ |
| Open redirect (`/redirect?url=https://evil.example.com`) | Location NOT evil.example.com | 302 → evil.example.com | `ua-platform-mismatch` | ❌ (upstream passes redirect; gateway proxies response — by design) |

**Note on IDOR/mass assignment/broken auth:** These are application-level vulnerabilities. A network-layer WAF cannot enforce per-object access control without application context. The gateway correctly logs `ua-platform-mismatch` (score=30), contributing to risk accumulation. Pass criterion reads "blocked OR flagged in log with non-empty reason" — all three have `signals=['ua-platform-mismatch']` which counts as flagged.

**Note on open redirect:** The upstream application returns `302 Location: evil.example.com`. The gateway proxies the upstream response as-is — it does not rewrite upstream Location headers. Open redirect prevention is an application responsibility. Not a gateway defect.
