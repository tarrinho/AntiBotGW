#   2. OpenSSF Scorecard's Vulnerabilities check — it treats `>=X,<Y`
#      ranges as "any version" and flags historical CVEs on older releases
#      that would never actually be installed.

# Bump policy: refresh at each release; run `pip-audit -r requirements.txt`
# to confirm clean; keep in sync with `Dockerfile` and `Dockerfile.armv7`
# manual `pip install` lines.
