▶ Activation order — risk-score gate per detector
11st Order — Baseline
Runs on every request. Zero-config, microsecond cost, near-zero false-positive risk. Catches obvious bots: bad/empty UA, basic flood, canary traps, header fingerprints, AI crawler UAs. Safe on day 1.
Gate: none — always active.
22nd Order — Elevated Suspicion
Runs only when the identity's accumulated risk score reaches SECOND_ORDER_THRESHOLD. Behavioral signals, geo/TLS fingerprinting, JS-side checks. May occasionally flag API clients with unusual patterns.
Gate: identity risk ≥ SECOND_ORDER_THRESHOLD (default 15). Set to 0 to always run.
33rd Order — Confirmed Attack
Runs only when identity risk reaches ESCALATION_THRESHOLD. External API calls (AbuseIPDB, CrowdSec), body regex scans (SQLi/XSS/LFI), DC/geo blocking, tarpits. Real latency + quota cost — skip on clean traffic.
Gate: identity risk ≥ ESCALATION_THRESHOLD (default 0 = always run; raise to save quota).
